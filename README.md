# Chinese_Chess_robot

基于 DK-2500 与 SCARA 机械臂的中国象棋对弈机器人

## 项目概述

本项目设计并实现了一台**人机对弈的中国象棋机器人**，系统整合了**机器视觉**、**人工智能决策**、**机械臂运动控制**、**网络通信**与 **Qt 图形界面**五大模块。机器人支持中国象棋标准规则下的执红/执黑对弈，并提供悔棋、一键摆盘、替玩家思考（AI 辅助落子）、残局对弈、语音播报等功能。此外，机器人还可进行五子棋对弈。

- **项目时间**：2026.04 – 2026.06
- **主控平台**：DK-2500 开发板
- **机械结构**：SCARA 机械臂（闭环步进电机驱动）

## 功能特性

| 功能 | 说明 |
|------|------|
| 🎮 **人机对弈** | 支持执红/执黑，完整的中国象棋标准规则 |
| ↩️ **悔棋** | 对弈过程中可随时悔棋，机械臂自动恢复局面 |
| 🔄 **一键摆盘** | 自动识别并复位所有棋子到初始位置 |
| 🤖 **替玩家思考** | AI 辅助落子——玩家点击按钮后，引擎计算最优走法并由机器人自动执行 |
| ♟️ **残局对弈** | 支持从任意残局局面开始对弈 |
| 🔊 **语音播报** | 走法语音播报，提升交互体验 |
| 🎯 **五子棋模式** | 机器人同时支持五子棋对弈 |

## 各子系统详解

### 1. 视觉系统

- **图像采集**：USB 摄像头实时拍摄棋盘画面
- **透视变换**：使用 OpenCV 对棋盘区域进行透视矫正，消除拍摄角度带来的畸变
- **棋子检测与分类**：
  - 使用 `HoughCircles` 圆检测定位棋子位置
  - 使用 **YOLOv8** 分类模型识别棋子类型（将/士/象/马/车/炮/兵 × 红/黑共 14 类）
- **棋盘矩阵构建**：
  - 将棋盘划分为 **10×9 网格**
  - 遍历所有检测到的棋子，填充完整的 10×9 棋子矩阵
  - 转换为中国象棋 **FEN 格式**状态字符串
- **鲁棒性增强**：
  - **帧间投票**：多帧检测结果投票，减少单帧误检
  - **棋子数量校验**：验证红黑双方棋子数量是否合法
  - **预期局面对比**：与上一回合预期局面进行格式比对，过滤异常检测结果

### 2. 人类走法提取与合法性校验

- **走法提取**：通过玩家落子前后局面的 FEN 格式，利用**差分算法**推导出玩家的 UCCI 走法（如 `b2e2`）
- **合法性校验**：遍历当前局面下所有合法走法列表，逐一比对：
  - 若玩家走法在合法列表中 → 接受走法，轮到 AI 回合
  - 若玩家走法**不在**合法列表中（如马别腿、将帅对面、蹩脚等违规走法）→ 判定为**非法走法**
- **非法走法处理**：控制机械臂将棋子恢复到玩家落子前的位置，并通过界面提示玩家"不能这样走"

### 3. 引擎决策与安全机制

- **引擎选型**：采用 **Pikafish** 象棋引擎作为 AI 决策核心（UCCI 协议兼容）
- **通信方式**：通过**子进程**与主程序通信，实时发送局面 FEN 及历史走法，接收引擎计算的最佳走法
- **难度调节**：通过设置单步思考时间（`movetime`）控制 AI 棋力水平
- **替玩家思考**：在人类回合，玩家点击按钮后，引擎为当前局面计算一步最优走法，由机器人自动执行落子
- **安全机制**：
  - 走法合法性二次校验
  - 引擎异常超时保护
  - 局面状态一致性检查

### 4. 机械臂控制

- **坐标映射**：将引擎走法的 UCI 坐标（如 `b2e2`）通过**线性映射**转换为透视图像素坐标
- **通信协议**：经 **TCP** 将目标像素坐标发送至机械臂客户端
- **手眼标定**：客户端基于手眼标定矩阵，将视觉坐标转换到机械臂基座坐标系
- **运动规划**：
  - 通过**运动学逆解**求得各关节目标角度
  - 驱动**闭环步进电机**完成精确拾放动作
  - 控制末端执行器（吸盘/夹爪）完成抓取与释放
